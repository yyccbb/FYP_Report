\section{Preliminaries}

\textsc{Definition 1.} \textbf{\textit{Road Network.}} A road network is represented as a directed graph consisting of intersections $\mathcal{V}$ and lanes $\mathcal{L}$. The lane set is grouped into three categories: go-through lanes ($\mathcal{L}_{\mathrm{go}}$), left-turn lanes ($\mathcal{L}_{\mathrm{left}}$), and right-turn lanes ($\mathcal{L}_{\mathrm{right}}$). Each lane connects adjacent intersections and is partitioned into segments $S = \{s_1, \ldots, s_n\}$ according to its distance from the intersection.

\textsc{Definition 2.} \textbf{\textit{Signal Phase.}} At each signal-switching step, the controller of an intersection chooses a signal phase from a predefined action set $A = \{a_1, \ldots, a_m\}$. A phase is written as $a = \mathrm{set}(\mathcal{L}_{\mathrm{allow}})$, where $\mathcal{L}_{\mathrm{allow}}$ denotes the collection of lanes that are allowed to move simultaneously without conflict, meaning these lanes receive green signals while conflicting lanes remain red.

\textsc{Definition 3.} \textbf{\textit{Traffic Signal Control System.}} The traffic signal control system consists of multiple agents $\Pi = \{\pi_1, \ldots, \pi_n\}$, with each agent assigned to one of the $n$ intersections in the road network. For intersection $i$, agent $\pi_i$ makes its signal control decision by observing the number of vehicles in each segment of the lanes approaching that intersection, thereby forming a segment-level view of the local traffic state.

\textsc{Problem Statement.} \textbf{\textit{LLM Reasoning for Traffic Signal Control.}} For each intersection $i$ in the road network, an LLM serves as agent $\pi_i$. At decision step $t$, the agent takes the observation $O_{i,t}$, the task description $d_{\mathrm{task}}$, and the action space $A$ as input, and outputs both reasoning content and a signal phase decision. The TSC problem is therefore formulated as selecting a phase $a_{i,t} \in A$ while generating reasoning output $R_{i,t}$ under the given observation and task specification. The system dynamics are defined as:

\begin{equation}
(R_{i,t}, a_{i,t}) = \pi_{\theta_i}(O_{i,t}, d_{\mathrm{task}}, A)
\end{equation}

where $O_{i,t}$ is the observation, $d_{\mathrm{task}}$ is the task description, $A$ is the action space, $R_{i,t}$ is the reasoning output, and $a_{i,t}$ is the selected phase.
